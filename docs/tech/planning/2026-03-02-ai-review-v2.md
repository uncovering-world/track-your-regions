# AI Hierarchy Review v2 — Implementation Plan

**Status:** Implemented

**Goal:** Add rename/reparent tree controls, make AI reports persistent with structured action checklists, and update AI prompts to output actionable JSON alongside markdown.

**Architecture:** Backend adds two new endpoints (rename, reparent) following existing controller patterns. AI prompts changed to request JSON with `report` + `actions` fields. Frontend stores reports in a React state Map keyed by region, renders actions as checkboxes in the persistent drawer.

**Design doc:** `docs/tech/planning/2026-03-02-ai-review-v2-design.md`

## What was built

1. **Rename & Reparent endpoints** — `POST .../rename-region`, `POST .../reparent-region` with Zod validation, world-view ownership checks, and circular-reference prevention (recursive CTE)
2. **Frontend mutations** — `renameMutation`, `reparentMutation` in `useTreeMutations.ts` with tree invalidation
3. **Rename button & dialog** — Edit icon on all tree nodes, Dialog with TextField and Enter support
4. **Reparent button & dialog** — DriveFileMove icon on non-root nodes, Dialog with Autocomplete dropdown of all regions (depth-indented)
5. **Persistent review reports** — `Map<string, StoredReport>` keyed by `"full"` or `"region-{id}"`, survives drawer close/reopen, with Regenerate button
6. **Structured AI output** — Backend prompts request JSON `{ report, actions }`, graceful degradation to plain markdown
7. **Actions checklist** — Checkboxes with type chips, radio groups for multi-choice actions, completion counter in footer

## Future improvements

- Wire action checkboxes to actually execute the corresponding tree mutation (rename, reparent, remove, etc.) with a single click
- Persist reports across page refreshes (localStorage or server-side)
- Batch-apply all uncompleted actions in one operation
