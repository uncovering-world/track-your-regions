
- **Run syncs** — trigger and monitor data synchronization from external sources (UNESCO, Wikidata for museums, landmarks)
- **Manage world views** — full CRUD on regional hierarchies, assign divisions to regions, compute geometries. When splitting GADM divisions into children, existing child regions (e.g., from a WorldView import) appear as assignment targets — admins manually assign which GADM geometry goes to which existing region via dropdowns. The "Create Custom Subregions" dialog also pre-populates target groups from existing child regions, letting admins drag divisions into imported regions. The Map View tab highlights the image overlay button when a region map is available — one click loads the map from Wikimedia Commons as a reference image, viewable either as a map overlay or side-by-side with the division map for precise geometry alignment. The dialog can be opened for any region — when a region has child regions but no direct divisions, descendant geometries are shown as a read-only context layer (color-coded by group) while only direct members remain interactive. A "Move to parent" tool lets admins click individual divisions to push them up to the parent region, and an "All to parent" batch action moves all unassigned divisions at once. A **CV auto-match** pipeline can automatically match color-coded map images to GADM divisions using K-means clustering and ICP alignment — the pipeline pauses for admin review of detected clusters, and if automated clustering produces incorrect results, admins can switch to a **vector border editor** to fix region boundaries (erase wrong borders, draw new ones with endpoint snapping) and fill regions with cluster colors via flood fill
- **Import WorldView** — two import paths: (1) click "Fetch from Wikivoyage" to automatically extract the full Wikivoyage region hierarchy (~5,700 regions), enrich with Wikidata IDs, create a WorldView, and match countries to GADM divisions — all from a single button click with live multi-phase progress (extraction → enrichment → import → matching, 20-40 min). Pages with ambiguous region entries (multi-link names, plain-text groupings) are automatically resolved via AI extraction when OpenAI is configured — the parser flags ~7% of pages for AI, which replaces heuristic classification with structured JSON extraction. When the AI is uncertain, it queues a question for admin review using a Human-in-the-Loop interview system: the admin sees one structured question at a time with clickable option buttons and an AI recommendation. Answers are processed to generate **generic rules** that improve ALL future extractions (e.g., "Cities are always leaf nodes"), not just the specific page. Page existence is verified against the MediaWiki API and shown in the preview; (2) upload a pre-generated JSON file for non-Wikivoyage sources. Online imports use a persistent cache — a "Use cached data" checkbox shows cache size and age, letting admins skip re-fetching unchanged pages or force a clean fetch. Each import path gets a distinct source type (`wikivoyage` vs `imported`) shown in the existing world views list. Admins choose a matching policy per import: "Country-based" (auto-match countries to GADM, the default) or "None" (skip auto-matching, all regions start unmatched for manual assignment). After extraction, a **hierarchy review** step flags regions where the parser dropped sub-regions (missing Wikivoyage pages, empty groupings) — a warning banner shows the count and admins can navigate flagged nodes via prev/next in the toolbar. Each flagged node offers "AI Review" (reads cached Wikivoyage page content to suggest missing children), "Add Child" (manual), or "Dismiss" to mark as reviewed. Review/accept matches through a dedicated admin interface with a hierarchical tree view. The tree view uses role-based rendering: containers show "X/Y matched" summaries, countries show status chips with GADM names, and `children_matched` countries expand to show subdivision assignments. Sub-continental groupings (Melanesia, Polynesia, etc.) can be handled via a "Handle as sub-continental" button that clears the parent's match and re-runs country-level matching on each child independently. Division assignments made via the WorldView Editor are automatically reflected back in the match review interface (bidirectional sync). A "Re-match All" button (with policy selector) resets and re-runs the matcher after improvements. "Close Review" finalizes the import (disabled until all regions are matched)
- **AI Settings** — centralized control over AI model selection per feature (matching, hierarchy review, Wikivoyage extraction, extraction interview, subdivision assist) with a usage dashboard showing per-session token counts, costs, and aggregated totals (today/month/all-time). Model choices are persisted in the database with a 60-second in-memory cache. A "Learned Rules" section shows user-provided rules that get injected into AI prompts — corrections and patterns that improve future extractions, added manually or generated automatically from interview answers
- **Manage curators** — promote users to curator role, assign scopes (region, source, or global), revoke assignments
- **Region assignment** — run the spatial assignment algorithm that maps experiences to regions based on their coordinates
- **Monitor system health** — sync logs, error tracking, database status

## The Experience System

Experiences are the atomic units of travel engagement — anything a user can discover, learn about, or do in connection with a region. Three experience categories are live: UNESCO World Heritage Sites (~1,250), Museums (~100 top art museums), and Public Art & Monuments (~200). Plans to expand into books, films, food, festivals, wildlife, and many more categories.
